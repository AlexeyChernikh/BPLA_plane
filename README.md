# Планировщик миссий QGroundControl

Desktop-приложение загружает большой Polygon/MultiPolygon и GeoTIFF DEM,
автоматически выбирает точки взлёта, делит участок на зоны обслуживания и
создаёт батарейные waypoint-миссии QGroundControl.

## Установка

Требуется Python 3.11 или новее:

```powershell
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Запуск

Двойным щелчком по `launch.bat` или командой:

```powershell
py -3 app/main.py
```

## Работа с приложением

1. Укажите метрическую рабочую CRS.
2. Загрузите SHP, GeoJSON, GPKG или KML.
3. Загрузите GeoTIFF DEM, полностью покрывающий полигон.
4. Настройте AGL, азимут, шаг профилей, батарею и скорости набора/снижения.
5. Нажмите **Найти Home и сгенерировать**.
6. При необходимости перетащите найденные Home на карте.
7. Проверьте зоны, миссии и предупреждения, затем выполните экспорт.

После расчёта кнопка **Сохранить проект** создаёт переносимый файл `.bpla`.
В него входят полигон, DEM, настройки расчёта, координаты Home и размеры сетки.
Также сохраняются уже рассчитанные зоны, профили, маршруты и высоты waypoint.
Кнопка **Открыть проект** восстанавливает готовый результат без повторного
расчёта, после чего его можно сразу просматривать и экспортировать. Проекты
старого формата автоматически пересчитываются при первом открытии.

Программа подбирает максимально крупный безопасный блок регулярной сетки,
чтобы уменьшить число Home. Для каждого блока выбирается плоская площадка,
а профили обрезаются границами малых батарейных миссий.

Крупные зоны Home и малые зоны миссий строятся по общей квадратной сетке,
повёрнутой по азимуту профилей. Внутренние ячейки имеют одинаковый размер,
а крайние обрезаются границей участка. Поля максимальной стороны зоны и
миссии ограничивают автоматический расчёт; `0` означает автоматический режим.
Азимут отсчитывается от истинного севера; сближение меридианов рабочей CRS
компенсируется автоматически.

На карте доступна древовидная легенда. Можно скрыть DEM, исходный полигон,
Home, зоны, миссии, профили и маршруты, а также отдельно отключить любую зону
или миссию.

Высоты waypoint вычисляются по DEM с постоянной AGL. Доступны режимы:

- Calc Above Terrain — режим по умолчанию: абсолютный профиль DEM + AGL
  со сглаживанием по скоростям набора и снижения;
- абсолютная AMSL;
- относительно Home;
- MAVLink Terrain Frame.

Режимы `Survey` и `Calc Above Terrain` выбраны по умолчанию и повторяют структуру
приложенного примера: Camera Mode, Takeoff, ComplexItem Survey и RTL. При
необходимости можно выбрать обычную последовательность Waypoint и другой
режим высот.

Перепад рельефа выше заданного порога выводится как предупреждение и не
блокирует расчёт. Перемещение Home, после которого часть зоны становится
недостижимой, блокирует экспорт до исправления.

## Результаты

Экспорт создаёт иерархию:

```text
zones/
  zone_001/
    zone_polygon.kml
    home.kml
    mission_001/
      zone_001_mission_001.plan
      zone_001_mission_001_grid.kml
      zone_001_mission_001_polygon.kml
      zone_001_mission_001_route.kml
      zone_001_mission_001_summary.csv
summary/
  all_mission_polygons.kml
  missions_map.png
  takeoff_points.gpkg
  operational_zones.gpkg
  mission_zones.gpkg
  all_profiles.gpkg
  missions_preview.geojson
  mission_summary.csv
```

Каждая папка миссии содержит QGC-план, KML её сети профилей, малого полигона,
полного маршрута и отдельную таблицу параметров.
